Politique de confidentialité

Dernière mise à jour : 12/03/2025

1. Collecte et utilisation des données

Cette extension, Automatisation Vote PandakMC, ne collecte aucune donnée personnelle auprès des utilisateurs. Les seules informations utilisées sont les suivantes :

  • Identifiant unique du CPU : L'extension génère un identifiant unique basé sur le modèle du CPU de l'appareil. Cet identifiant est utilisé uniquement pour vérifier que la clé d'activation est utilisée sur un seul appareil. Aucune information personnelle n'est collectée ou partagée.
  • Clés d'activation : Les clés d'activation sont vérifiées via une API GitHub pour s'assurer qu'elles sont valides et non utilisées sur plusieurs appareils. Aucune donnée personnelle n'est transmise à des tiers.
  • Données locales : L'extension utilise le stockage local (chrome.storage.local) pour enregistrer des informations telles que l'état d'activation de la clé, les délais de vote (nextVoteTimeSite1 et nextVoteTimeSite2), et d'autres paramètres nécessaires au bon fonctionnement de l'extension. Ces données ne quittent jamais l'appareil de l'utilisateur.

2. Partage des données

Cette extension ne partage aucune donnée avec des tiers. Les informations utilisées (comme l'identifiant unique du CPU et les clés d'activation) sont traitées localement ou via une API GitHub sécurisée pour vérifier la validité des clés. Aucune donnée n'est transmise à des serveurs externes ou à des tiers.

3. Permissions utilisées

L'extension utilise les permissions suivantes pour fonctionner correctement. Voici les justifications pour chaque permission :

  • activeTab : Cette permission permet à l'extension d'interagir avec les onglets ouverts sur les sites de vote (PandakMC, Serveur-Privé, et Top-Serveurs). Elle est utilisée pour cliquer sur les boutons de vote et naviguer entre les pages.
  • alarms : Cette permission est utilisée pour planifier les votes à intervalles réguliers (1h30 ou 2h). Elle permet de respecter les délais imposés par les sites de vote.
  • identity : Cette permission est utilisée pour accéder à l'API GitHub afin de vérifier les clés d'activation. Elle permet de s'assurer que les clés sont valides et non utilisées sur plusieurs appareils.
  • scripting : Cette permission est utilisée pour exécuter des scripts sur les pages de vote. Elle permet de cliquer sur les boutons de vote, de récupérer les délais de vote, et d'interagir avec les éléments de la page.
  • storage : Cette permission est utilisée pour stocker des informations locales, comme l'état d'activation de la clé et les délais de vote. Ces données ne quittent jamais l'appareil de l'utilisateur.
  • system.cpu : Cette permission est utilisée pour générer un identifiant unique basé sur le modèle du CPU. Cet identifiant est utilisé pour vérifier que la clé d'activation est utilisée sur un seul appareil.
  • tabs : Cette permission est utilisée pour gérer les onglets ouverts sur les sites de vote. Elle permet de naviguer entre les pages et de fermer les onglets après le vote.

4. Sécurité des données

La sécurité des données est une priorité pour cette extension. Voici les mesures prises pour protéger les informations des utilisateurs :

  • Aucune collecte de données personnelles : L'extension ne collecte ni ne stocke aucune donnée personnelle.
  • Données locales : Toutes les informations sont stockées localement sur l'appareil de l'utilisateur via chrome.storage.local. Ces données ne sont pas transmises à des serveurs externes.
  • API sécurisée : Les clés d'activation sont vérifiées via une API GitHub sécurisée (HTTPS). Aucune donnée personnelle n'est transmise lors de cette vérification.

5. Objectif unique de l'extension

L'objectif unique de cette extension est d'automatiser les votes sur PandakMC pour les sites Serveur-Privé et Top-Serveurs. Elle permet aux utilisateurs de gagner du temps en votant automatiquement toutes les 1h30 ou 2h, selon les règles des sites. Une clé d'activation est nécessaire pour utiliser l'extension.

6. Utilisation de code distant

L'extension utilise du code distant pour vérifier les clés d'activation via une API GitHub. Cela permet de s'assurer que les clés sont valides et non utilisées sur plusieurs appareils. Aucune donnée personnelle n'est transmise lors de cette vérification.

7. Transparence et contrôle

Les utilisateurs ont un contrôle total sur les données utilisées par l'extension. Aucune donnée personnelle n'est collectée, et les informations stockées localement peuvent être supprimées à tout moment en désinstallant l'extension.
